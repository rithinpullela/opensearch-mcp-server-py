# Adversarial Review Log — Modular Rebuild

Running, append-only checkpoint log for the adversarial review of the `opensearch-mcp-server-py` modular rebuild (`main..HEAD`). Each entry is a zoomed-out, human-facing verdict on rebuild quality at a point in time. Newest entries are appended at the bottom.

## [checkpoint] — On track and faithful, but two ordering/parser defects and ~1100 LOC of inert scaffold need attention before wire-in  (P0-P3)

**Bird-eye OLD-vs-NEW.** The rebuild is on track and the actually-wired work is a genuine net simplification: deleting the runtime OpenAPI tool generator (boot-time unpinned GitHub fetch, no timeout, `print()`-into-JSON-RPC hazard) and replacing it with four golden-snapshot-locked static tools is the right trade, and extracting `compat.py` as a leaf cleanly breaks the real `tools→generic_api_tool→tool_filter→tools` import cycle without lazy-import dodges. Fidelity discipline is unusually strong — the request handler is a byte-for-byte port (NDJSON spacing, GET-with-body, plain `json.dumps`), the version-gate `'1.0'` semantics are preserved, and all 684 tests pass with zero outbound network at import. The main drag on the checkpoint is that it also lands ~1100 LOC of deliberately-UNWIRED machinery (settings/auth_strategy/version_cache/context/registry/modules) plus ~1700 lines of overlapping (and partly superseded) planning docs alongside the still-live originals, so none of those abstractions has earned its keep yet and several carry latent defects. Net: faithful and simpler where it touches production, but accumulating more "present but inert" surface than a "easy to contribute" mid-flight codebase should carry.

### What is genuinely good
- **`compat.py` is dependency inversion done well.** Moving `check_tool_compatibility` to a leaf taking injected `registry` + `version_fetcher` genuinely breaks the `tools→generic_api_tool→tool_filter→tools` cycle without lazy-import dodges; the 1-line delegator keeps ~40 call sites untouched, and the bare-`Exception` raise + exact message text are reproduced byte-for-byte (pinned by `test_compat.py`).
- **The handler port is verbatim, verified line-by-line against `main:src/tools/tool_generator.py`.** `process_body` (NDJSON spacing), `select_endpoint` (method-ignoring GET-with-body), base-arg extraction (missing→`''`), path substitution, and plain `json.dumps` (not the compact `format_json`) are character-for-character identical — a less careful port would have silently changed every response's separators/ensure_ascii.
- **The golden-snapshot lock is a strong runtime→static migration oracle.** Captures generator output before deletion and asserts dict-equality on `input_schema` plus byte-exact `display_name`/`description`/`http_methods`/`min_version('1.0')`/`max_version('99.99.99')`, including correctly NOT normalizing `'1.0'`→`'1.0.0'` and correctly OMITTING `required` when empty (vs emitting `required: []`).
- **Deliberate fidelity over "helpful" fixes.** The `/None/_explain/{id}` garbage-path quirk and loose `Optional[str]=None` arg models are preserved, not silently tightened — exactly right for a fidelity checkpoint.
- **Clean value objects and drop-ins.** `ServerContext` (frozen+slots dataclass mirroring each `global_state` field, with the `get_mode()`-defaults-to-`single` footgun diagnosed in its docstring); `ToolRegistry` (full dict read API, fail-loud `DuplicateToolError`, `as_dict()` shallow copy so deepcopying consumers are unaffected); `version_cache._normalize_url` (scheme/host lowercasing, default-port insertion, IPv6 bracketing, userinfo/trailing-slash stripping, injected monotonic clock for deterministic TTL tests).
- **`settings.py` per-variable fidelity is right in spirit.** It preserves inverted `OPENSEARCH_SSL_VERIFY` ('True unless "false"'), tri-state `OPENSEARCH_DYNAMIC_CONNECTION` left raw, and no-try/except `OPENSEARCH_TIMEOUT` that still raises — the correct instinct for a no-drift rebuild (modulo the truthy-union MAJOR above).
- **The `#271` `/mcp` no-307 fix is minimal and provably protocol-neutral.** `Route('/mcp')` + `Mount('/mcp')` with a tiny `_ASGIApp` wrapper; confirmed `StreamableHTTPSessionManager.handle_request` never inspects `scope['path']`, and `test_mcp_endpoint_no_redirect` asserts 200 with no `location` header.
- **Staying on low-level `Server`** (vs a FastMCP port) is correctly justified by per-mode schema mutation + SDK v2 churn, and the docstrings are unusually honest about each module's inert status and residual risk.

### Decisions to revisit (QUESTIONs for the human)
- **`required`-array ordering normalized to deterministic.** The static schemas hard-code `required` order (e.g. `['body','id','index']`); the old generator emitted set-iteration (non-deterministic) order. Benign and semantically equivalent (JSON-Schema treats `required` as a set), but clients now receive stable bytes where they previously did not. Confirm the captured order matches what real clients received and log it as a deliberate, benign normalization (DECISION_LOG D2/D7).
- **Frozen `min_version='1.0'` / `max_version='99.99.99'`.** The old code read `x-version-added`/`x-version-deprecated` from the live spec every boot; the static tools freeze these forever. Intended consequence of de-pinning (P0-1) and `is_tool_compatible` parses `'1.0'`≡`'1.0.0'`, so no gate math drift — but confirm this freeze of an upstream-tracking field is a conscious sign-off.
- **`auth_strategy` fail-secure raise is a real behavior change.** It RAISES on partial AWS creds (access/secret without the full triple) where `client.py` previously fell through to IAM/basic/ambient. This is the documented O4 decision and the right call, but the eventual wire-in PR must flag it in CHANGELOG/release notes as a breaking change for partial-credential configs.
- **`version_cache` proportionality.** 193 lines (negative-TTL floor, clock injection, IPv6-aware normalizer) for a per-call-version-fetch churn bug — and DESIGN_DECISIONS itself notes the cache does NOT fix the real correctness hole (version=None still enables 3.3.0+ tools). Confirm the churn is a measured problem before wiring this much machinery; a plain TTL dict keyed on URL may suffice for v1.
- **Sequencing of scaffold vs wire-in.** Is shipping P2 modules unwired-and-untested-against-live (then migrating in later phases) the intended cadence, or should each phase wire what it introduces? This governs how much latent drift risk the tree carries between checkpoints.

Decide the per-domain file shape now so `core.py`/`cat.py` don't inherit fragmentation that the cohesion doesn't warrant (≤~150-200 LOC domains likely want one file).

- **`ToolSpec.http_methods` required-vs-validated mismatch (above).** Decide whether `http_methods` is contractually required (then add to `_REQUIRED_KEYS`) given the write-protection filter depends on it.
