01

Why touch it at all

The server worked — but it carried a 1,297-line registry monolith, a runtime OpenAPI generator that fetched a spec from GitHub on every boot, and a handful of latent correctness bugs. The goal was a middle ground: reorganize for contributors and fix what's broken, without a ground-up rewrite that makes the diff unreviewable.

□

Correctness first

Then maintainability, then latency & memory. Behavior changes are deliberate and documented — never accidental drift.

≡

Minimal diff

Prefer surgical in-place edits over new abstractions. A reviewer should grok each change against the old code in under 30 seconds.

⟳

Test-gated

The ~30 integration tests are the 1:1 oracle. Every phase stayed green; tool names, schemas, output text, CLI, env, and endpoints are unchanged.

02

Old architecture → new

Before

  • 1,297-line tools.py — one giant TOOL_REGISTRY dict literal + 35 handlers
  • Runtime OpenAPI generator fetched a spec from an unpinned GitHub branch at every startup (no timeout, 30s+ hang offline, print() into the stdio JSON-RPC stream)
  • A fresh client + GET / on every single tool call just to gate the version
  • Tool errors reported as success — failures never set the MCP isError flag
  • Response-size limit off by default despite docs promising 10MB; a 404 could fire two HTTP requests
  • Partial AWS creds silently fell through to the server's ambient identity
→

After

  • 987-line tools.py — metadata in per-domain modules, composed via an explicit 5-group manifest
  • Generator deleted. 4 static tools, byte-locked to a golden snapshot. Zero network to build the catalog
  • Per-target version cache (TTL, async-safe) collapses the per-call probe; bypassed under header-auth
  • Failures set isError=true on the wire — verified end-to-end against the SDK
  • 10MB enforced by default, streamed with early abort; transport errors propagate, never re-issued
  • Partial creds fail secure (raise); URL credentials scrubbed from logs (CWE-532)
03

What actually changed

Five headline changes. Each ties back to a cited audit finding and a logged decision — nothing here is reorganization for its own sake.

Correctness · O1

The isError contract

Tool-execution failures (connection refused, index_not_found, parse errors, 4xx/5xx) now set CallToolResult.isError=true. Before, spec-compliant clients and LLM agents saw every error as a success.

Why: the #1 audit finding. Fixed at one chokepoint; exact error text preserved in content.
Structure · P0

Generator, deleted

The boot-time OpenAPI fetch is gone. Its 4 tools are now static, hand-written, and locked to a snapshot captured from the live generator before deletion. The server builds its 48-tool catalog with no outbound connection.

Why: a network dependency, a 30s offline hang, and a stdout write that corrupts the stdio protocol — all removed.
Efficiency · P1

Version-gate caching

The compatibility gate ran a fresh client + GET / on every call. Now a per-connection-target, TTL-bounded, async-safe cache serves it — with a per-request bypass when header auth makes the target dynamic.

Why: doubled network round-trips per tool call against a value that rarely changes.
Memory & safety

Response-size limiting

10MB enforced by default (matching the docs), checked incrementally while streaming so it aborts before buffering the whole body. The error-re-issuing fallback — which double-fired non-idempotent writes — was deleted.

Why: protection was silently off; the old fallback was a correctness hazard.
Security

Auth, hardened in place

Partial AWS header credentials now fail secure instead of borrowing the server's identity. URL userinfo is scrubbed from logs, the IAM ARN moved off INFO, and the bearer path no longer drops the custom User-Agent.

Why: privilege-confusion + CWE-532 log leakage. A 280-line "resolver" was built, reviewed, then deleted in favor of a +53/-9 surgical fix.
Structure · P1

The monolith, split

The 343-line registry literal became a typed, fail-loud ToolRegistry composed from per-domain modules in an explicit group order. Handlers stayed put — only metadata moved — so git-blame is preserved and the diff is ~30 lines of wiring.

Why: adding a tool meant editing a giant dict. Now each tool lives in a greppable module.
